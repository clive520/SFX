# SFX Lab Website Plan

## 目標

SFX Lab 是一個給創作者、直播主、影片剪輯者、遊戲主持人使用的網頁音效台。使用者打開網站後，可以快速播放長時間背景音效與短時間音效，並依照當下工作需要調整音量、排序常用音效。

## 核心使用情境

- 使用者需要在一個畫面裡快速播放常用音效。
- 使用者需要把常用音效拖到上方，減少臨場尋找時間。
- 使用者需要長時間播放背景音，例如雨聲、海浪、颱風、快樂音樂、悲傷音樂、沉重音樂。
- 使用者需要短時間觸發音效，例如 UI 點擊、金幣、雷射、轉場、撞擊、能量啟動。

## 目前功能

- 長時間背景音效區。
- 短時間音效區。
- 主音量滑桿。
- 主音量開關。
- 每個背景音可單獨調整音量。
- 背景音可播放與停止。
- 短音效可即時觸發。
- 音效卡片可拖曳排序。
- 排序會存在瀏覽器 localStorage。
- 卡片採緊湊顯示，說明文字 hover 時浮現。
- 部分背景音使用下載到本機的 Mixkit MP3。

## 技術方向

- 優先採用純靜態網站：HTML、CSS、JavaScript。
- 優先部署到 GitHub Pages，因為目前功能不需要後端。
- 音訊控制使用 Web Audio API。
- 真實音檔放在 `assets/audio/`，避免外部連結失效、CORS 或 hotlink 問題。
- 授權資訊記錄在 `CREDITS.md`。
- 每次完成可用功能後 commit，並同步到 GitHub。

## GitHub Pages 適用性

目前網站可在 GitHub Pages 正常運作，因為它只需要：

- 靜態 HTML/CSS/JS。
- 靜態 MP3 檔案。
- 瀏覽器 localStorage。
- 瀏覽器 Web Audio API。

## 可能需要 Firebase 的情況

若未來加入以下功能，GitHub Pages 可能不夠，需要轉到 Firebase 或其他後端：

- 使用者登入。
- 多裝置同步排序與音量設定。
- 使用者自行上傳音檔並永久保存。
- 後台管理音效素材。
- 統計播放次數。
- 多人共用同一組音效台設定。
